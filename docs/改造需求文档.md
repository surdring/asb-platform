# ASB-Platform × PolyBrowser Agent Platform 改造需求文档

> 将 PolyBrowser Agent Platform 的前端 UI 接入 ASB-Platform 真实后端 API，打造完整可用的浏览器采集平台产品。

---

## 一、项目背景与目标

### 1.1 现状

| 项目 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **ASB-Platform** | 后端引擎 | 真实 CDP 通信、SQLite 持久化、14 个 REST API、SSE 事件总线 | 无 UI，只能 curl 调用 |
| **PolyBrowser Agent Platform** | 前端 UI | React 19 + Tailwind 暗色主题、中英文切换、5 个管理页面 | 全部 mock 假数据，不能执行任何采集任务 |

### 1.2 改造目标

将 PolyBrowser Agent Platform 的前端代码改造为 ASB-Platform 的 **官方 Web 管理面板**，实现：

1. **数据真实化**：所有页面数据从 ASB-Platform REST API 实时获取，移除全部 mock 数据
2. **操作可用化**：页面上的按钮（创建环境、启动/停止、租用标签页、执行任务等）真正调用后端 API
3. **实时化**：通过 SSE 事件总线实现 Dashboard 状态实时更新、Broker 命令总线实时展示
4. **功能完整化**：补齐后端已有但前端缺失的功能（采集结果查看、日志查看、任务执行表单等）

## 七、技术方案

### 7.1 前端 API 客户端

```typescript
// src/api/client.ts
const BASE_URL = '/api'

async function request<T>(path: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE_URL}${path}`, {
    headers: { 'Content-Type': 'application/json' },
    ...options
  })
  if (!res.ok) {
    const err = await res.json().catch(() => ({ error: res.statusText }))
    throw new ApiError(res.status, err.error || 'Request failed')
  }
  return res.json()
}

export const api = {
  get: <T>(path: string) => request<T>(path),
  post: <T>(path: string, body?: unknown) =>
    request<T>(path, { method: 'POST', body: body ? JSON.stringify(body) : undefined }),
  delete: <T>(path: string) => request<T>(path, { method: 'DELETE' })
}
```

### 7.2 SSE 事件订阅 Hook

```typescript
// src/hooks/useSSE.ts
function useSSE(onEvent: (type: string, data: unknown) => void) {
  useEffect(() => {
    const source = new EventSource('/api/events')
    source.onmessage = (e) => {
      // SSE 事件格式: event: <type>\ndata: <json>\n\n
      // EventSource 默认触发 onmessage，需自行解析
    }
    // 监听自定义事件
    const events = ['environment.created', 'task.completed', 'lease.released']
    events.forEach(type => {
      source.addEventListener(type, (e) => {
        onEvent(type, JSON.parse(e.data))
      })
    })
    return () => source.close()
  }, [])
}
```

## 九、风险与注意事项

### 9.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SSE 兼容性 | EventSource 在某些浏览器/代理下可能断开 | 增加自动重连机制，断开后 3 秒重试 |
| 后端 API 变更 | 前端类型定义需同步更新 | API 类型集中管理在 `src/types/api.ts` |
| CORS 问题 | Vite 代理已解决同源问题，但直接访问后端端口可能跨域 | 后端已设置 `Access-Control-Allow-Origin: *` |

### 9.2 设计原则

1. **渐进改造**：先让页面"能看数据"，再让按钮"能操作"，最后加实时推送
2. **保持 UI 风格**：不改变现有的暗色赛博朋克主题、布局结构、动画效果
3. **错误友好**：API 失败时展示错误提示而非白屏，保留最后一次成功数据
4. **零额外依赖**：不引入 axios/swr/react-query 等，使用原生 `fetch` + 自定义 Hooks

### 9.3 兼容性说明

- 前端开发时需确保 ASB-Platform 后端已在 `:8787` 启动
- 如果后端不可用，页面应展示"后端服务未连接"状态，而非崩溃
- 所有 API 调用需有超时处理（默认 10 秒）

## 十、验收标准

1. **Dashboard**：展示真实的环境数、租约数、技能数、任务数；最近任务列表为真实数据
2. **Environments**：能查看、创建、启动、停止浏览器环境
3. **Broker**：能查看、创建、释放标签页租约；命令总线展示 SSE 实时事件

5. **Tasks**：能查看任务历史；能通过表单选择租约+技能+动作执行新任务
6. **Collected Items**（新增）：能查看采集结果列表
7. **Logs**（新增）：能查看系统日志，支持筛选
8. **实时通知**：任务完成/失败时弹出通知
9. **中英文切换**：所有新增文案支持中英文
10. **错误处理**：后端不可用时前端展示友好提示，不崩溃
