       ## 对比分析：ASB-Platform vs PolyBrowser Agent Platform

### 一、定位差异

| 维度 | **ASB-Platform**（本目录） | **PolyBrowser Agent Platform**（GEMINI STUDIO） |
|------|---------------------------|----------------------------------------------|
| **本质** | 真实可运行的后端引擎 | 前端 UI 概念演示（Demo/Mockup） |
| **目标** | 提供完整的浏览器采集基础设施 | 展示平台管理面板的 UI 交互概念 |
| **可运行性** | ✅ 完整可运行，`npm start` 启动真实服务 | ⚠️ 仅前端 UI，数据全部为 mock 假数据 |

---

### 二、技术栈对比

| 维度 | **ASB-Platform** | **PolyBrowser Agent Platform** |
|------|-----------------|-------------------------------|
| **运行时** | Node.js ≥ 22 | Node.js + Vite |
| **语言** | 纯 JavaScript（ES Module） | TypeScript + React (TSX) |
| **外部依赖** | **零依赖**（纯 Node.js 内置模块） | 15+ npm 依赖 |
| **前端框架** | 无（纯 API 服务） | React 19 + Tailwind CSS 4 |
| **HTTP 服务** | `node:http` 原生模块 | Express + Vite dev server |
| **数据库** | SQLite（`node:sqlite`） | 无（内存 mock 数据） |
| **浏览器协议** | CDP WebSocket 直连 | 无（仅 UI 展示） |
| **AI 集成** | 无 | `@google/genai`（Gemini API） |
| **国际化** | 无 | `i18next` + `react-i18next`（中/英） |
| **动画** | 无 | `motion`（Framer Motion） |
| **图标** | 无 | `lucide-react` |
| **Markdown** | 无 | `react-markdown` |

### 四、架构对比

**ASB-Platform** 是真正的分层后端架构：

```
HTTP API → Broker → TaskRunner → CDP Client → Chrome/Edge
                → LeaseManager
                → SkillRegistry
                → SQLiteStore → asb.sqlite
```

**PolyBrowser Agent Platform** 是纯前端 SPA：

```
React App → mock 数据（store.ts 中的硬编码数组）
         → 5 个页面组件（Dashboard/Environments/Broker/Skills/Tasks）
         → 无后端、无数据库、无浏览器交互
```

### 六、总结

| | **ASB-Platform** | **PolyBrowser Agent Platform** |
|------|------|------|
| **一句话** | 能跑的后端引擎 | 好看的 UI 概念图 |
| **优势** | 真实可用、零依赖、完整功能 | 视觉精美、交互流畅、国际化 |
| **劣势** | 无 UI 界面，只能 curl 调用 | 全是假数据，不能真正执行任何采集任务 |

**两者关系**：PolyBrowser Agent Platform 本质上是 ASB-Platform 的 **UI 概念原型**——它用 mock 数据模拟了 ASB-Platform 中环境管理、租约管理、技能库、任务调度等核心概念的 UI 呈现方式。如果要把 ASB-Platform 做成一个有 UI 的产品，PolyBrowser Agent Platform 的 UI 设计是一个很好的参考方向
        
            
       
总结来说：

- **ASB-Platform** 是"能跑的后端"——零依赖、真实 CDP 协议通信、SQLite 持久化、完整的 REST API，但没有任何 UI
- **PolyBrowser Agent Platform** 是"好看的 UI 概念图"——React + Tailwind 暗色主题、中英文切换、5 个管理页面，但所有数据都是硬编码的 mock，不能真正执行任何采集任务

两者恰好互补：一个缺 UI，一个缺后端。如果需要，可以把 PolyBrowser Agent Platform 的 UI 作为参考，为 ASB-Platform 接入真实 API 数据，打造一个完整的产品。
